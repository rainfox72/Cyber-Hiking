# v2.5 Balance Adjustment + AI Auto-Play — Design Document

## Problem

### Balance Issues (from 20-game automated playtest)
- **100% defeat rate** across all bot strategies (aggressive, cautious, escape)
- **Hypothermia is universal killer**: body temp averages 8% at death
- **Starting body temp too low** (50%) — players begin in a deficit
- **Water surplus at death** (2.4L remaining) — hydration drains faster than consumption allows
- **Passive altitude drain too punishing** — energy bleeds away above 3000m
- **Camp recovery insufficient** — especially body temp, which never recovers enough to offset drain

### Missing Feature: AI Auto-Play
- Ollama is already integrated for narration but only used post-turn
- Players want to watch the AI play the game — "spectator mode"
- Ollama can make strategic decisions based on game state context

## Part 1: Balance Adjustment

Eight specific number changes to make the game survivable without trivializing it:

| Parameter | Current | New | Rationale |
|-----------|---------|-----|-----------|
| Starting body temp | 50% | 70% | Players shouldn't start cold |
| Camp body temp recovery (shelter) | +8/hour-mult | +12/hour-mult | Camping at shelter should meaningfully warm you |
| Camp body temp recovery (no shelter) | +3/hour-mult | +5/hour-mult | Even exposed camping should help |
| Rest body temp recovery | +3 | +5 | Resting should warm you slightly |
| Passive altitude energy drain | -1/hour above 3000m | -0.5/hour above 3000m | Halved — altitude is punishing enough via O2 |
| Weather force multiplier (blizzard/wind) | Energy -3, BodyTemp -3 | Energy -2, BodyTemp -2 | Reduce weather lethality without removing danger |
| Camp fatigue (2nd consecutive camp) | 35% recovery | 50% recovery | Diminishing returns too harsh |
| Hydration drain (push_forward) | -15 | -10 | Players die with water in their pack |

## Part 2: AI Auto-Play

### Architecture
- New `src/services/ollamaDecision.ts` — `generateDecision(state, validActions) → { action, reasoning }`
- Sends game context (location, vitals, weather, inventory, available actions) to Ollama
- Ollama returns JSON with chosen action + 1-sentence reasoning
- 2 Ollama calls per AI turn: decision (pick action) then narration (describe outcome)

### Decision Prompt Template
```
You are an AI mountaineer on the Ao Tai Line (鳌太线). Based on your current situation, choose the best action.

Current state:
- Location: {waypoint name} ({elevation}m)
- Weather: {condition}, {intensity}%
- Time: Day {day}, {hour}:00
- Energy: {energy}%, Hydration: {hydration}%, Body Temp: {bodyTemp}%
- O2 Sat: {o2}%, Morale: {morale}%
- Food: {food} meals, Water: {water}L, Gear: {gear}%, Medicine: {medicine}
- Lost: {isLost}

Available actions: {action list}

Respond with JSON: { "action": "action_id", "reasoning": "one sentence why" }
```

### Auto-Play Loop
1. User clicks "AUTO" toggle button in Navigation Console
2. Store sets `autoPlayEnabled: true`
3. Loop: wait 3-4 seconds → call `generateDecision()` → `performAction()` → repeat
4. Stops when: game ends (victory/defeat), user clicks AUTO again, or Ollama disconnects
5. 15-second timeout on decision call → fallback to cautious heuristic bot

### Fallback Heuristic Bot
When Ollama is disconnected or times out:
- If energy < 30 and food > 0: eat
- If hydration < 30 and water > 0: drink
- If bodyTemp < 30 and can camp: set_camp
- If energy < 40 and can camp: set_camp
- Otherwise: push_forward (or descend if at point-of-no-return waypoint)

### UI Changes

#### Navigation Console
- "AUTO" toggle button at top of action list
- When active: action buttons dimmed (not clickable), "AI THINKING..." indicator pulses
- AUTO button changes color: off = dim outline, on = cyan/neon blue

#### Log Window
- AI reasoning entries: `[AI] {reasoning}` prefix
- Cyan/blue text color to distinguish from green narration and amber events

#### Store
- `autoPlayEnabled: boolean` — toggle state
- `autoPlayLoop()` — async function managing the decision→action→wait cycle
- `stopAutoPlay()` — cancels loop
