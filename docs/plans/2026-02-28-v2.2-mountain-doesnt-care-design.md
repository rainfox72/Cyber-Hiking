## Section B: Action System Overhaul (8 Actions)

**Removed:** WAIT (folded into REST for "hunker down")

| # | Action | Duration | Effect | Change |
|---|--------|----------|--------|--------|
| 1 | PUSH FORWARD | 3-5h | Move forward, rolls lost + fall checks | New: lost/fall rolls |
| 2 | SET CAMP | **Smart: 4h day / until-dawn night** | Recovery scales with hours. canCamp only | **Redesigned** |
| 3 | DESCEND | 2h | Retreat 1 waypoint. Blocked when lost | New: blocked when lost |
| 4 | CHECK MAP | 1h | Core navigation mechanic. Reduces lost, boosts find-way-back | **Overhauled** |
| 5 | REST | 2h | +8 energy, +3 bodyTemp. Also "wait out weather" | Absorbs WAIT role |
| 6 | EAT RATION | 0.5h | +25 energy, +5 morale, 10% poison | No change |
| 7 | DRINK WATER | 0.5h | +30 hydration, +3 morale | No change |
| 8 | USE MEDICINE | 0.5h | O2 +15 & temp normalize, OR fall recovery (+25% all vitals) | **New: fall mode** |

### Smart Camp Detail
- **Before 17:00**: 4h camp, same recovery as v2.1
- **After 17:00**: Camps until 06:00 (variable 4-13h)
- Recovery formula: `base * (hours/4) * fatigueMult * resourceMult`
  - Example: 20:00-06:00 (10h) = `15 * 2.5 = +37.5 energy` (fresh, fed)
  - Hydration drain: `-2/h * 10h = -20 hydration` (burns water fast)
  - Smart players eat+drink before camping

### CHECK MAP Overhaul
- Still costs 1h, 3 energy
- Sets `checkedMapThisSegment = true` (resets after each push_forward)
- When lost: increases `findWayBackChance` by +25%
- When on trail: gives -5% lost chance bonus for next push

### When Lost — Action Availability
- PUSH FORWARD: allowed (wander, rolls find-way-back)
- CHECK MAP: allowed (boost find-way-back)
- REST, EAT, DRINK, MEDICINE: allowed
- SET CAMP: blocked (no campsite off-trail)
- DESCEND: blocked (don't know where trail is)

## Section C: Getting Lost System

New hidden navigation accuracy stat governs whether hiker stays on-trail.

### New PlayerState Fields
```typescript
isLost: boolean           // currently off-trail
lostTurns: number         // turns spent lost
lostFromWaypointIndex: number  // where they went off-trail
checkedMapThisSegment: boolean // reset after each push
findWayBackChance: number // current % to find trail (0-100)
```

### Getting Lost Probability (per push_forward)

| Factor | Effect |
|--------|--------|
| Base chance | 10% |
| Didn't check map | +15% (25% total) |
| Check map bonus | -5% (5% total) |
| Night time | x1.5 multiplicative |
| Fog weather | +20% additive |
| Blizzard | +15% additive |
| Ridge/scree terrain | +5% additive |
| Forest terrain | -10% additive |

### Finding Way Back

| Factor | Effect |
|--------|--------|
| Base find-way-back | 15% |
| Each check_map while lost | +25% cumulative |
| Night time | x0.5 (halved) |
| Fog | x0.7 |

When found: place hiker at random waypoint within +/-2 of where they got lost.

### Probability Analysis
- Smart player (checks map): 5% lost per push, ~1 lost episode per run
- Careless player (no map): 25%+ per push, ~3-4 lost episodes
- Each lost episode: 2-3 turns of check_map to recover (40%/65%/90%)

## Section D: Fall/Drop System

Falls are a new critical event type triggered on push_forward.

### Fall Probability (per push_forward)

**Base by terrain:**

| Terrain | Base Fall % |
|---------|------------|
| Stream valley | 2% |
| Forest | 2% |
| Meadow | 2% |
| Summit | 4% |
| Ridge | 5% |
| Scree | 6% |
| Stone sea | 8% |

**Multiplicative modifiers:**

| Condition | Multiplier |
|-----------|-----------|
| Blizzard | x1.5 |
| Night | x1.5 |
| Wind | x1.3 |
| Rain/snow | x1.2 |
| Currently lost | x1.3 |
| Energy < 30% | x1.3 |

### Fall Damage
- **50% reduction** to ALL vitals (energy, hydration, bodyTemp, O2, morale)
- **Instant death** if average health <= 25% at time of fall
- Gear: -20%
- 30% chance to scatter 1 food or 0.5L water

### Fall Recovery
- USE MEDICINE with `fall_injury` status effect: recover +25% all vitals, clear effect
- Without medicine: damage is permanent
- `fall_injury` is a StatusEffect: { id: "fall_injury", duration: Infinity, ... }

### Expected Falls Per Run
- Over ~18 turns, avg 3-5% per push = 0.5-0.9 falls
- Most runs: 0-1 falls. One fall survivable with medicine. Two = death.

## Section E: Terrain + Weather + Night Stacking

All three systems stack multiplicatively.

### Terrain Danger Profiles

| Terrain | Lost Modifier | Fall Base | Notes |
|---------|--------------|-----------|-------|
| stream_valley | -15% | 2% | Safe early terrain |
| forest | -10% | 2% | Landmarks help |
| meadow | +0% | 2% | Open, moderate |
| ridge | +5% | 5% | Exposed, wind-vulnerable |
| scree | +5% | 6% | Ankle-twisting |
| stone_sea | +10% | 8% | Worst — loose rocks |
| summit | +5% | 4% | Thin air, exposed |

### Weather Amplifiers (multiplicative)

| Weather | Lost x | Fall x | Extra |
|---------|--------|--------|-------|
| Clear | 1.0 | 1.0 | — |
| Cloudy | 1.0 | 1.0 | — |
| Fog | 1.2 (+20% add) | 1.2 | Visibility near zero |
| Rain | 1.1 | 1.2 | Wet rocks |
| Snow | 1.2 | 1.2 | Trail obscured |
| Wind | 1.1 | 1.3 | Blown off balance |
| Blizzard | 1.15 (+15% add) | 1.5 | Maximum danger |

### Night Multiplier
- Lost: x1.5
- Fall: x1.5
- Stacks with everything

### Worst Case: Ridge + Blizzard + Night
- Lost (no map): (25% + 5% + 15%) x 1.5 = 67%
- Lost (with map): (5% + 5% + 15%) x 1.5 = 37%
- Fall: 5% x 1.5 x 1.5 = 11%

## Section F: Map Visualization When Lost

- Hiker marker drifts off trail to random offset position
- Pulsing "?" icon appears above hiker
- Trail ahead becomes hidden (future dashed lines disappear)
- Reddish/fog tint overlay on map
- When found: marker animates back to waypoint with snap effect

## Section G: Win Rate Math (~30%)

Skilled run: 12 pushes + ~2 lost episodes (4 extra turns) + 50% chance of 1 fall:
- 60% survive lost system (2 episodes, find way back each time)
- 90% survive fall system (55% no fall + 35% fall and survive with medicine)
- 55% survive resource management (food/water/vitals over 16-18 turns)
- Combined: 0.6 x 0.9 x 0.55 ~ **30%**
