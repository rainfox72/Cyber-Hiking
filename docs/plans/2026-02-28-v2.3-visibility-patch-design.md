# v2.3 "Visibility Patch" — Design Document

**Date:** 2026-02-28
**Version:** v2.3
**Status:** Approved

## Overview

Five UX/gameplay fixes addressing readability, death feedback, lost probability, lost map clarity, and hiker visual redesign.

---

## Issue 1: Vignette Too Dark

### Problem
When worst vital drops below 30%, the vignette (tunnel vision overlay) becomes so dark that buttons, log text, and Ollama narration are unreadable. Players cannot interact with the game during critical moments.

### Current Values
- `worstVital > 60`: no vignette
- `worstVital > 30`: radius 70%
- `worstVital > 10`: radius 40%
- `worstVital <= 10`: radius 20%
- Outer edge: `rgba(0,0,0,0.85)`

### Solution: Cap vignette intensity

**New thresholds:**
- `worstVital > 60`: no vignette (unchanged)
- `worstVital > 30`: radius **80%** (was 70%)
- `worstVital > 10`: radius **60%** (was 40%)
- `worstVital <= 10`: radius **40%** (was 20%)

**Reduce max opacity:** Outer edge from `rgba(0,0,0,0.85)` to `rgba(0,0,0,0.6)`.

Tint colors (cold blue, low O2 red) stay unchanged.

### Files
- `src/components/effects/Vignette.tsx`

## Issue 2: Death Cause Not Shown Before Death Screen

### Problem
When a vital hits 0, the game instantly jumps to the SIGNAL LOST defeat screen. Players don't understand what killed them — especially for falls where vitals are still nonzero but averaged below 25%.

### Solution: "Dying breath" intermediate state

Add a `"dying"` game phase that shows the cause for ~2 seconds before transitioning to defeat.

**Flow:**
1. Vital hits 0 OR fatal fall → set `gamePhase = "dying"`, `dyingCause = "..."`
2. GameOverlay renders a red flash overlay with the cause text prominently displayed
3. After 2 seconds, automatically transition to `gamePhase = "defeat"`

**Dying causes (map to existing defeat strings):**
- Energy 0: "EXHAUSTION — BODY SHUTDOWN"
- Hydration 0: "DEHYDRATION — ORGAN FAILURE"
- Body Temp 0: "HYPOTHERMIA — FROZEN"
- O2 Sat 0: "ALTITUDE SICKNESS — OXYGEN FAILURE"
- Morale 0: "DESPAIR — WILL BROKEN"
- Fatal fall: "FATAL FALL — THE MOUNTAIN CLAIMS YOU"

**UI:** Full-screen dark red overlay (`rgba(80,0,0,0.9)`) with large monospace cause text, pulsing, centered. No buttons — auto-transitions after 2s.

### Files
- `src/engine/types.ts` — add `"dying"` to GamePhase union, add `dyingCause` field
- `src/engine/gameEngine.ts` — set `gamePhase = "dying"` instead of `"defeat"` in fatal conditions
- `src/engine/vitalCalculator.ts` — `checkDefeatCondition` returns structured cause
- `src/components/game/GameOverlay.tsx` — render dying phase overlay, auto-transition to defeat
- `src/store/gameStore.ts` — handle dying→defeat transition

## Issue 3: Getting Lost Probability Too High

### Problem
Without checking map, lost chance is 25% (base 10% + no-map 15%) before modifiers. Players feel they get lost too frequently, especially early in the game.

### Solution: Reduce base + weaken penalties

| Parameter | Current | New |
|-----------|---------|-----|
| `BASE_LOST_CHANCE` | 0.10 | **0.06** |
| `NO_MAP_CHECK_PENALTY` | 0.15 | **0.10** |
| `MAP_CHECK_BONUS` | -0.05 | **-0.04** |

**Net effect:**
- Without map check: **16%** (was 25%) before terrain/weather
- With map check: **2%** (was 5%) before terrain/weather
- Map check provides 8x safety bonus, strongly encouraging the mechanic

### Files
- `src/engine/navigationSystem.ts` — update 3 constants

## Issue 4: Lost State Not Clear on Map

### Problem
When lost, only a small pulsing "?" appears above the hiker and a very faint red tint (`rgba(120,0,0,0.08)`) covers the map. These are too subtle to notice.

### Solution: Bold "LOST" label + stronger visual indicators

1. **Replace "?" with "LOST" label** — Larger red text below the hiker marker, pulsing animation, `fontSize="10"`, bold monospace
2. **Increase red tint** — From `rgba(120,0,0,0.08)` to `rgba(120,0,0,0.15)`
3. **Dashed search radius ring** — Red dashed circle (`r="20"`) around the hiker that pulses, representing the "search area" while lost

### Files
- `src/components/map/HumanMarker.tsx` — replace "?" with "LOST" label
- `src/components/map/TacticalMap.tsx` — increase tint opacity, add search radius ring
- `src/App.css` — update lost-pulse animation if needed

## Issue 5: Cyberpunk/Glitch Hiker Redesign

### Problem
The hiker is a basic stick figure (lines + circles) that doesn't match the cyberpunk terminal aesthetic.

### Solution: Geometric glitch silhouette with scanline effects

**Body redesign:**
- Replace stick-figure lines with filled geometric silhouette using `<polygon>` and `<path>` elements
- Trapezoid torso, angular limbs, distinct angular backpack shape
- All 8 poses redesigned in this geometric style

**Glitch effects:**
- 2-3 thin horizontal scanline `<rect>` elements across the figure with flickering opacity animation
- Scanlines animate at different rates for organic feel

**Data glow redesign:**
- Replace circular pulse ring with a hexagonal or diamond-shaped "scan field"
- Stroke-dasharray animation for rotating dash effect

**Health degradation visual:**
- As health drops below 30%, scanline intensity increases (more visible)
- Below 10%, add slight horizontal jitter animation (corrupting signal)

**Poses to redesign (all 8):**
1. Idle — standing with backpack, slight lean
2. Walking — stride pose, trailing arm holds pole
3. Camping — crouching by tent/shelter shape
4. Eating — seated, arm to face
5. Drinking — standing, canteen raised
6. Resting — leaning against backpack
7. Mapping — arms extended holding map rectangle
8. Medicine — kneeling, first-aid cross visible

### Files
- `src/components/map/HumanMarker.tsx` — complete rewrite of all pose components
- `src/App.css` — scanline, jitter, and scan-field animations
