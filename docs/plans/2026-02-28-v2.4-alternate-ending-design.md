# v2.4 Alternate Ending — Design Document

## Problem

The game has only one win condition: reach Baxian Platform (waypoint 12). Players who realize the mountain is too dangerous have no incentive to turn back — descending is purely a retreat mechanic with no payoff.

## Solution

Add a second victory condition: descending back to Tangkou (waypoint 0) after venturing out on the trail. This creates two distinct endings:

- **Ending 1: Escape** — Return to the trailhead alive. Bittersweet/survival tone.
- **Ending 2: Summit** — Reach Baxian Platform. Triumphant tone (existing).

## Escape Win Condition

Triggers when ALL of these are true:
- `currentWaypointIndex === 0` (player is at Tangkou)
- `turnNumber >= 4` (player has been on the trail long enough)
- Action was `descend` (arrived by descending, not game start)

The `turnNumber >= 4` guard prevents trivial wins. By turn 4, the player has made meaningful decisions on the trail.

## Engine Changes

### `types.ts`
- Add `endingType: "escape" | "summit" | null` to `GameState` interface

### `gameEngine.ts`
- `createInitialState()`: set `endingType: null`
- `cloneState()`: copy `endingType`
- Step 9 (win check): add escape check before summit check
  - If `action === "descend"` AND `newState.player.currentWaypointIndex === 0` AND `newState.turnNumber >= 4`: set `gamePhase = "victory"`, `endingType = "escape"`, push escape log entry
  - Existing summit check: add `endingType = "summit"`

### Validation
No changes needed. Descending to waypoint 0 is already valid (`currentWaypointIndex > 0` check passes from waypoint 1). Point-of-no-return (waypoint 11+) still blocks descend, so escape is only reachable from waypoints 1-10.

## Store Changes

### `gameStore.ts`
- Add `endingType: "escape" | "summit" | null` to `GameStore` interface
- Add to `stateFromGameState()` and `gameStateFromStore()` mappings
- Reset to `null` in `startGame()`

## Victory Screen Changes

### `GameOverlay.tsx`
Branch on `endingType`:

**Ending 1: Escape**
- Amber color scheme (not green)
- Title: `ESCAPE`
- Subtitle: `ENDING 1 // YOU SURVIVED`
- Stats: Days, Distance, Turns (amber values)
- Flavor text: `You turned back and lived. The mountain remains unconquered.`
- Amber glow-pulse effect

**Ending 2: Summit** (existing, with minor additions)
- Green color scheme (unchanged)
- Title: `SUMMIT REACHED` (unchanged)
- Subtitle: `ENDING 2 // THE ROOF OF QINLING`
- Stats: unchanged
- Flavor text: unchanged
- Green glow-pulse (unchanged)

### `App.css`
- `.screen-overlay--escape` class with amber background/glow
- Amber glow-pulse keyframes
- Amber stat value color

## What Stays the Same
- No waypoint data changes
- No navigation system changes
- No new game phases — both endings use `"victory"` phase
- No sound changes (victory sound plays for both)
- Point-of-no-return (waypoint 11+) unchanged
- Defeat/dying mechanics unchanged
