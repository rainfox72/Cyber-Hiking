Implementation Approach: Two Strategies

Given the scale (11 phases, ~30+ files), how should we sequence and structure this overhaul?

A

Foundation-First: Art Direction → Layout → Scene → Polish (Recommended)

Sequence:

  1. Art direction core — new CSS variables, palette, typography, panel styling (Phase A + I merged)
  2. Layout restructure — full-bleed mountain layer, floating panels, z-index stack refactor
  3. Mountain scene system — SVG ridgelines, terrain bands, sky gradients, time-of-day (Phase B)
  4. Atmosphere canvas — fog, snow, rain, lightning, blizzard over the scene (Phase E upgrade)
  5. Hiker upgrade — expressive poses, condition-reactive animations (Phase C)
  6. Action juice + danger telegraphing — micro-feedback, crisis visuals (Phase D + F)
  7. Screens overhaul — title, death, victory, run summary (Phase G + J)
  8. Route identity — waypoint visual differentiation on map (Phase H)
  9. Difficulty retune — engine balance to <10% (Phase K)

Philosophy: Build the visual foundation first, then layer expressiveness on top. Each phase builds on the previous one's infrastructure. The mountain scene exists before we add weather drama to it. The palette exists before we add danger telegraphing.

Risk: Later phases depend on earlier ones being solid. If the foundation needs iteration, downstream work shifts.

Strength: Coherent visual system from the start. No orphaned effects. Each phase has a stable base to build on.

B

Impact-First: Scene → Weather → Hiker → Then Polish Everything

Sequence:

  1. Mountain scene + layout — biggest visual change first, prove the concept (Phase B + layout)
  2. Weather drama — make weather threatening and experiential (Phase E)
  3. Hiker expressiveness — bring the character to life (Phase C)
  4. Art direction + UI premium pass — unify everything after the big pieces exist (Phase A + I)
  5. Action juice + danger telegraphing — layer on feedback (Phase D + F)
  6. Screens + route identity + replay — polish sequences (Phase G + H + J)
  7. Difficulty retune — final balance pass (Phase K)

Philosophy: Ship the most dramatic changes first. See the mountain scene working, get excited, then refine everything around it. Delay systematic palette/UI work until the big pieces are in place.

Risk: Early work may use ad-hoc styling that needs cleanup later. Possible rework when the unified art direction lands.

Strength: Fastest path to "wow." Can evaluate the mountain scene early and course-correct.

My recommendation: Option A

Foundation-first avoids rework. With this scope, building on sand means rebuilding later. If we establish the palette, typography, and panel system first, every subsequent component automatically inherits the visual language. The mountain scene renders with the right colors from day one. Weather effects use the right opacity/contrast hierarchy immediately. Danger telegraphing plugs into existing CSS variable states.

The risk of "later phases shifting" is low because your brief is specific enough that the foundation design is clear — we're not guessing.

Estimated scope: ~25-35 files modified/created across 9 implementation phases. New components: MountainScene, AtmosphereCanvas, terrain profile data, danger overlay system, run summary screen. Major refactors: App layout, all panel components, HumanMarker, ParticleCanvas, GameOverlay, TitleScreen, CSS variables.
