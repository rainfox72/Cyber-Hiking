Art Direction: Mountain Scene Rendering Approach

The mountain scene (Phase B) is the single biggest visual addition. How should we render it?

A

Layered SVG Silhouettes (Recommended)

Technique: 4-6 SVG mountain ridge layers with parallax CSS transforms, animated cloud/fog layers, CSS gradient skies that shift with time-of-day and weather. Waypoint terrain changes swap layer shapes and colors.

Pros: Crisp at any resolution. Lightweight. Easy to animate with CSS transforms. Natural fit with existing SVG map. Strong shape language. Can be dramatic with good composition.

Cons: Limited organic detail. Must rely on silhouette design skill rather than texture.

Vibe: Alto's Adventure meets tactical command screen. Clean, stark, atmospheric.

B

Canvas 2D Painted Scene

Technique: Dedicated <canvas> element rendering layered terrain with gradient fills, procedural texture, and particle integration. All painting happens in a render loop alongside existing particle canvas.

Pros: More painterly atmosphere. Can do dynamic lighting, glow, and texture effects. Particles and scene in one unified rendering context.

Cons: Heavier rendering budget. More complex code. Harder to maintain. Risk of looking muddled at wrong scale.

Vibe: Darkest Dungeon background meets radar screen. Moody, textured, immersive.

C

Hybrid: SVG Silhouettes + Canvas Atmosphere

Technique: SVG layers for mountain shapes and structure, canvas overlay for atmospheric effects (fog banks, lightning, blowing snow, light rays). Best of both worlds but more complex integration.

Pros: Sharp geometry + rich atmosphere. Flexible. Can progressively enhance.

Cons: Two rendering systems to coordinate. More code. Potential z-index headaches.

Vibe: Expedition terminal with a living viewport. The mountain breathes through the instruments.

Context: Where the scene goes

Currently the center panel holds: location info → tactical map → log window → weather/day-night. The mountain scene would sit behind or above the tactical map as a backdrop, making the map feel like it's overlaid on the actual mountain landscape. The hiker marker would visually connect both layers.

All three approaches support waypoint-specific terrain variation, time-of-day sky changes, and weather-reactive visuals. The question is about rendering fidelity and production cost.
