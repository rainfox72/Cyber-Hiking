Design Section 3: Hiker + Action Feedback + Danger Telegraphing

Making the hiker feel human, actions feel consequential, and danger feel readable.

Hiker Upgrade — From Marker to Person

Expressive Hiker System

The hiker stays as a geometric SVG figure (current style works), but gains condition-reactive modifiers layered on top of the 8 action poses.

Condition Modifiers (additive, stack on any pose):

Condition Visual Modifier Trigger
Cold Hunched shoulders (polygon shift), breath vapor particles (2-3 small circles animating upward from head, fade out). Blue tint on figure outline. bodyTemp < 50
Wind Figure leans into wind direction (slight rotate transform). Backpack strap / jacket edge flutter (small animated line segment). Hair/hood motion. weather = wind | blizzard
Exhaustion Lowered head position, slower bob animation (1.2s instead of 0.6s), slight forward lean. Walking stride shortens. energy < 35
Injury Visible limp in walk cycle (asymmetric bob). One arm held closer to body. Movement stutter. statusEffects includes fall_injury
Night Headlamp cone: small triangular gradient projecting forward from head position. Warm amber glow. Cone sways slightly with movement. timeOfDay = night | dusk
Critical Figure flickers (opacity stutter 0.4-1.0, irregular timing). Scan field distorts. Color shifts to desaturated. The "signal" is breaking up. any vital < 15

Special Moment Poses:

  • Summit pose: Arms raised, figure stands tall, brief golden glow pulse. Only appears on reaching waypoint 12. Should feel earned and rare.
  • Stumble recovery: Brief 300ms animation when a fall event occurs — figure drops, catches self, pauses. Plays before the injury modifier kicks in.
  • Lost wandering: Figure drifts off the trail path with uncertain micro-movements. Facing direction becomes inconsistent. Headlamp sweeps wider at night.

Implementation: Condition modifiers are SVG sub-elements toggled by state. Breath particles are 2-3 <circle> elements with CSS animation. Wind lean is a transform: rotate() on the figure group. Headlamp is an SVG <polygon> with gradient fill. No new rendering context needed — everything stays in the existing SVG layer.

Action Feedback — Every Decision Feels Consequential

Action Juice System

A new <ActionFeedback> component + CSS classes triggered by lastAction state changes. Each action gets a brief, punchy visual response.

Action Scene Response UI Response Feel
Push Forward Parallax layers shift (terrain traversal). Dust/snow puff from hiker feet. Brief directional motion streak. Route line pulses forward. Map micro-pans. Location text transitions. Commitment, momentum
Set Camp Scene dims slightly (settling in). Warm amber glow appears at hiker position (campfire). Particles slow. Panel borders soften briefly. Action area contracts to camp-relevant options. Amber wash on panels. Fragile safety, relief
Check Map Brief tactical grid overlay flashes on mountain scene (thin lines, fast fade). Map zooms slightly. Map panel border brightens. "Certainty" indicator pulses green. Route line refreshes with sweep animation. Tactical clarity
Fall / Injury Sharp screen shake (200ms, high intensity). Hiker stumble animation. Brief red flash on mountain layer. All panel borders flash red. Risk meter spikes. Vitals that dropped pulse danger. Impact sound. Shock, cost
Get Lost Scene blurs momentarily (200ms). Mountain layers shift in conflicting directions (disorientation). Fog thickens briefly. Map flickers. Waypoint markers jitter. Route line becomes uncertain (dashed/faded). Compass spin on location info. Disorientation, dread
Rest Scene breathing — very subtle zoom pulse (0.5%). Particles drift lazily. Calm. Vitals bars animate recovery smoothly. Panel glow dims to minimum. Brief respite
Eat / Drink Minimal scene change. Hiker pose animation. Relevant vital bar pulses green on recovery. Inventory counter decrements with brief flash. Satisfying tick. Resource spent wisely
Use Medicine Brief warm pulse around hiker (healing indicator). Status effect indicator fades with relief animation. Affected vitals pulse positive. Costly relief

Implementation: A useActionFeedback hook watches lastAction and lastEvents, then sets CSS classes and triggers brief animations. Most effects are CSS transitions/animations with JS-controlled class toggling. Screen shake reuses the existing isShaking mechanism but with configurable intensity.

Danger Telegraphing — Visual Crisis Language

Condition-Specific Visual Overlays

Upgrades the current Vignette into a <DangerOverlay> system with condition-specific visual treatments. Multiple overlays can stack.

Danger Screen Treatment UI Treatment
Hypothermia Blue-white frost creep from screen edges (CSS radial gradient). Breath emphasis on hiker. Scene desaturates toward cold blue. Frost crystal overlay at corners. Body temp bar ice-blue glow. Panel borders tint blue. Text gets slightly colder color temperature.
Altitude Sickness Subtle tunnel vision (vignette tightens). Slight pulsing desaturation synced to a slow "heartbeat" rhythm (1.2s). Peripheral darkening. O2 bar pulses with heartbeat. Risk meter text slightly blurs on pulse peaks. Faint reddish tint on vignette.
Dehydration Scene contrast harshens — blacks get blacker, highlights get harsher. Faint static/grain overlay. Dry, brittle visual feel. Hydration bar flickers. Panel borders get thin hairline cracks (SVG overlay). Text rendering feels slightly degraded (letter-spacing jitter).
Starvation Scene dims progressively. Colors drain. Movement feels heavier (slower parallax response). Energy bar dims. All panel text slightly reduced contrast. Action buttons feel heavier (opacity reduction on non-essential actions).
Morale Collapse Micro-jitter on scene layers (1-2px random displacement, intermittent). Scene feels unstable. Existing vitals jitter intensifies. Log text occasionally glitches (brief character substitution effect via CSS). Panel borders flicker inconsistently. The instruments feel unreliable.
Navigation Uncertainty Mountain layers shift subtly in conflicting directions. Fog thickens. Depth cues degrade. Waypoint markers on map pulse with uncertainty. Route line ahead becomes dashed/faded. "UNCERTAIN" label on location display.
High Fall Risk Foreground ridge layer shows steep drop-off detail. Vertigo — very subtle downward drift on near layers. Route segment ahead pulses red on map. Risk meter border glows. Terrain hazard icon appears near location info.

Stacking: Multiple conditions combine. A cold, exhausted, lost hiker at night shows frost edges + dimmed scene + navigation uncertainty + headlamp. The visual story compounds — the player sees the run collapsing through layered visual degradation.

Implementation: <DangerOverlay> component reads all player vitals and status effects, renders appropriate CSS overlays (positioned absolute, pointer-events: none). Each overlay is a separate div with its own animation. Component memoizes on relevant vital thresholds to avoid unnecessary re-renders.

Does the hiker expressiveness, action juice, and danger telegraphing feel right? The hiker gains condition modifiers that stack on poses. Actions get scene + UI responses. Danger creates layered visual degradation that tells the story of a collapsing run. Next: title/death/victory screens, route identity, replay drama, and difficulty tuning.
