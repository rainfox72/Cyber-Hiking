Design Section 2: Mountain Scene + Atmosphere + Weather

The SVG ridgeline system, terrain bands, sky states, and canvas atmosphere layer.

Mountain Scene Architecture

SVG Ridgeline Layer System

The mountain scene is a <MountainScene> component rendering a full-viewport SVG with 5 parallax ridge layers:

Layer 5 (farthest) — Distant range silhouette, lightest opacity (0.15-0.25)
                      Barely visible peaks, sets depth. Subtle parallax shift.

Layer 4             — Secondary ridgeline, medium-light (0.25-0.35)
                      Defines the broader mountain chain. Slow parallax.

Layer 3             — Primary ridge, medium opacity (0.4-0.55)
                      The dominant mountain profile. This is "the Ao Tai spine."
                      Shape changes per terrain band (5 profile sets).

Layer 2             — Near terrain, higher opacity (0.55-0.7)
                      Rocky outcrops, forest edges, scree slopes.
                      Most terrain variation here.

Layer 1 (nearest)  — Foreground detail, highest opacity (0.7-0.85)
                      Ground-level rock, grass tufts, snow drifts.
                      Frames the bottom edge of the viewport.

Each layer is a single SVG <path> with a filled polygon. Shapes are defined as data arrays — not hand-drawn SVG files. A terrainProfiles.ts data file holds path point arrays for each terrain band.

Parallax: Layers shift horizontally by transform: translateX(offset) based on waypoint progress. Farther layers move less. Creates a sense of traversal without actual scrolling.

5 Terrain Bands — Waypoint-Specific Profiles
Band Waypoints Elevation Visual Character
Forest Approach 0-2 (塘口→都督门) 1740-2600m Rounded ridges, dense treeline silhouettes on near layers, warmer color temperature. Ridges have organic soft contours. Foreground shows forest canopy shapes.
Rocky Spine 3-5 (老庙子→2800营地) 2800-3400m Jagged angular ridges, exposed rock faces. Treeline drops off — near layers become bare scree and broken stone. Cooler color shift. Sharper silhouette edges.
Exposed Plateau 6-8 (金字塔→九重石海) 3400-3600m Wide open ridgelines, long exposed traversals. Dramatic sky proportion — layers compress vertically, more sky visible. Wind-swept, desolate. Color desaturates.
Storm Ridge 9-11 (雷公庙→跑马梁) 3450-3600m Narrow knife-edge ridges, steep drop-offs implied by near layers cutting sharply. Darkest, most oppressive profiles. Cloud layers wrap close. Foreground detail is sparse — exposed rock only.
Summit Approach 12 (拔仙台) 3767m Single dominant peak silhouette rising above all layers. Other ridges fall below. Near layers show the final rocky scramble. Sky opens up — if weather permits, a rare golden/clear moment. If weather is bad, the summit is barely visible through whiteout.
Sky System — Time of Day × Weather

Sky is a CSS linear-gradient on the base layer, transitioned smoothly (2-3s) on time/weather changes.

Time Clear Sky Overcast/Storm
Dawn Deep indigo → warm peach at horizon → pale gold band Dark gray-blue → muted orange smear → heavy gray
Morning Steel blue → lighter blue-gray → pale horizon Uniform slate gray, slightly lighter at horizon
Midday Medium blue-gray → lighter gray (high altitude haze) Flat gray, oppressive ceiling feel
Afternoon Warm blue → golden-gray haze at horizon Darkening gray, amber-tinged at base
Dusk Deep purple → crimson → burnt orange band Dark bruised purple → dull red smear
Night Near-black → deep navy → faintest blue at horizon Pure black → charcoal. No horizon. Void.

Ridge layer colors also shift with time-of-day — dawn warms them slightly, night darkens and desaturates them, storm conditions push them toward uniform dark gray.

Atmosphere Canvas — Weather as Adversary

Canvas Atmosphere Layer (replaces ParticleCanvas)

A new <AtmosphereCanvas> component replaces the current ParticleCanvas. Same position (full viewport, pointer-events: none), but dramatically expanded capabilities. Renders between SVG mountains and UI panels in the z-stack.

Weather Modes:

Weather World Layer (on scene) UI Interference Layer
Clear Minimal — faint high-altitude haze, occasional dust mote. The mountain is crisp and visible. None. Clean instruments.
Cloudy Slow-moving cloud bank shapes drift across mid-layers. Partial ridge obscuration. None.
Fog Dense gradient fog rising from valleys. Far ridges vanish. Near layers fade. Depth cues collapse. Subtle haze on panel edges. Reduced contrast at panel boundaries.
Rain Angled streak particles across full viewport. Darkened surface tint. Puddle-shimmer effect on foreground layer. Occasional streak across panel glass. Screen surface texture (faint noise overlay).
Snow Moderate snowfall particles. Accumulation tint on foreground ridges (white edge). Reduced visibility at distance. Snow particles drift across panel edges. Frosting at corners in severe cold.
Wind Directional horizontal streak particles. Debris specks. Cloud layers move faster. Ridge shimmer on exposed layers. Panel edges flicker. Slight directional vibration on text (CSS, not canvas).
Blizzard Maximum particle density + directional chaos. Near-total ridge obscuration. Whiteout gradient washes out scene. Only closest foreground ridge partially visible. Heavy interference. Panel glass frosts. Text flicker on exposed panels. Static noise artifacts on risk meter. The instruments are fighting to stay readable.

Storm Lightning: During storm/blizzard weather, occasional lightning flashes: canvas fills with brief white flash (50ms), followed by scene silhouette momentarily lit in stark contrast, then slow fade (300ms). Frequency: 1-3 per turn during storms. Creates dramatic punctuation without constant noise.

Does this mountain + atmosphere system feel right? 5 terrain bands with distinct profiles, 6 time-of-day sky states × weather modifiers, and a weather system that escalates from atmospheric to actively interfering with the UI.
